👓 Copilot Lens

Copilot Lens

A local memory layer for your AI coding assistants. Copilot Lens parses the logs that Copilot CLI, VS Code Copilot Chat, and Claude Code already save on your machine, and makes them searchable, browsable, and measurable.

Browse every conversation

The tab lists every chat you have had with any of the three tools, side by side. Filter by source, date, status, or working directory. Click a session to replay it: messages in order, tool calls inline, and any thinking blocks expanded on demand.

Search across your whole history

Full-text search runs over every indexed session. Type in the search bar on and results rank by relevance with the matched text highlighted. Useful when you remember solving something a few weeks ago and just need to find it again.

Understand how you actually use these tools

The tab has eight charts: sessions per day, hour of day, tool usage, model usage, top directories, time per branch, time per repo, and MCP servers. A source filter at the top applies to everything on the page so you can compare Copilot CLI vs Claude Code at a glance.

See where tokens go

The tab shows daily, weekly, and monthly totals of input, output, and cached tokens, split by model. It also shows a rough cost estimate based on public API list prices. Curious how the numbers are built and what "cached" means? Read the tab.

Score each repo's effectiveness

The tab computes a per-repository score based on session outcomes, tool patterns, and time spent. A quick way to spot the codebases you are shipping in vs. the ones you are grinding through.

Export a session for training

Open any session from the tab and click ⬇ Export JSONL in the header. You get one line of OpenAI-style chat JSONL, ready for supervised fine-tuning with TRL, Axolotl, or OpenAI's fine-tuning API.

Your data stays on your machine

Copilot Lens reads ~/.copilot/logs and ~/.claude/projects directly. No sign-in, no cloud, no upload. Everything on every tab is parsed locally and stays on your disk.

How tokens are counted

When you chat with an AI coding tool, everything it sees (your message, the files it read, the rules it was given, the whole conversation so far) gets turned into "tokens" before the model runs. Every reply is also made of tokens. Copilot Lens reads your local logs and adds these numbers up.

A rough rule: 1 token is about 4 English letters, or ¾ of a word. A 200-line TypeScript file is usually 1,500 to 3,000 tokens.

What Copilot CLI records

GitHub Copilot CLI saves one log per day under ~/.copilot/logs. Every AI response has a usage block with three numbers:

prompt_tokens
Everything the model read this turn, cached or not.
completion_tokens
Everything the model wrote back.
cached_tokens
The part of prompt_tokens that came from cache, not fresh.

Copilot itself does not bill you per token. It sells a subscription with a monthly budget of "premium requests". What you see here is the underlying model usage that Copilot makes on your behalf.

What Claude Code records

Claude Code saves one JSONL file per session under ~/.claude/projects/<folder>/<session>.jsonl. Every assistant message has a usage block with four numbers:

input_tokens
Brand new input the model had to fully process.
output_tokens
What the model wrote back.
cache_creation_input_tokens
Input that was processed and also saved to cache for later. Billed at about 1.25x the normal input rate.
cache_read_input_tokens
Input pulled back from cache this turn. Billed at about 0.1x the normal input rate.

So the Copilot and Claude shapes are different. To compare them side by side, Copilot Lens folds the four Anthropic numbers into the same three the Copilot side uses:

input + cache_creation + cache_read
Cached
cache_read
Completion

Why "cached" matters

An AI assistant's context includes the system instructions, the whole conversation so far, and any files it has read. That can easily be tens of thousands of tokens, and it only grows as the session continues.

Sending all of that to the model on every turn would be slow and expensive. So the provider caches the stable part (system prompt, early messages, unchanged file contents) and reuses it the next time. You still pay for those tokens, but at roughly 10% of the normal input price.

The raw counters still count cached tokens as input, because the model still "sees" them. That is why a long Claude Code session can show 20 million input tokens even though almost all of it was cache reads at the discounted rate.

If you want to see only the fresh content the model actually had to work through, flip Cached: Exclude on the Tokens page. That subtracts the cached portion from Prompt and Total.

About the cost estimate

The Est. API Cost number on the Tokens page uses the public list prices from each provider and multiplies them by the tokens we saw in your logs. It is a rough "what if you had called the API yourself" figure, not your actual bill.

Caveats:

  • It is not what GitHub Copilot charges. Copilot sells a subscription and uses a premium request budget; that budget is cheaper than calling the underlying APIs directly.
  • It is not what Claude.ai or ChatGPT subscriptions charge. Same reason.
  • Cache creation is billed higher than cache reads. Copilot Lens approximates this but does not match to the exact penny.
  • Unknown model names count as zero cost. The "% priced" label tells you how much of the total had a known rate.

Where the data lives

~/.copilot/logs/
Copilot CLI response logs.

Every number on these pages is parsed from those files on your own machine. Nothing leaves your computer.
